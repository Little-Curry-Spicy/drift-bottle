# 漂流瓶 MVP（Expo + React Native）

这是一个可直接体验的漂流瓶前端 MVP，用于快速验证产品方向与核心交互。

## 当前实现的功能

- 扔瓶子：输入内容、选择心情标签、发布到海域
- 罗盘抛投：在扔瓶页可拖拽罗盘与纸飞机，模拟抛出方向与角度
- 蓄力反馈：拖拽距离越大，纸飞机越大，形成蓄力感
- 陀螺仪微动：设备轻微晃动会带来罗盘层的微动效果
- 捞瓶子：从陌生人瓶子中随机抽取
- 回复瓶子：对捞到的瓶子发送回复
- 收藏瓶子：将喜欢的瓶子加入收藏列表
- 我的瓶子：查看自己扔出的瓶子
- 数据看板：展示已扔出、已收藏、总回复数量

> 当前为本地状态模拟，无后端依赖；刷新应用后数据会重置。

## 运行方式

1. 安装依赖

```bash
pnpm install
```

2. 启动项目

```bash
pnpm start
```

## Clerk 鉴权配置（登录/注册）

1. 在 Clerk 创建 Expo 应用，获取 Publishable Key
2. 在项目根目录创建 `.env`
3. 写入以下变量：

```bash
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxx
```

配置后将启用以下页面：

- `/`：落地页
- `/sign-in`：登录页
- `/sign-up`：注册页（邮箱验证码）
- `/bottles`：漂流瓶主页面（需登录）

## 目录说明

- `app/index.tsx`：漂流瓶 MVP 主页面（核心交互）
- `app/_layout.tsx`：路由布局
- `global.css`：主题变量与全局样式

## MVP 交互说明

- 默认进入海域页，点击“开始捞瓶”会随机展示陌生人瓶子
- 在“扔瓶”页输入内容后点击“扔进海里”，会进入“我的”页看到新瓶子
- 在“扔瓶”页可在罗盘区域手指拖动，实时查看抛投角度与方向
- 在海域中可对当前瓶子进行“发送回复”与“收藏”
- “收藏”和“我的”页分别查看对应列表

## 下一步建议（可继续实现）

- 接入后端 API：用户体系、瓶子列表、回复与收藏持久化
- 增加审核能力：敏感词过滤、举报、拉黑
- 增加推荐逻辑：按标签与活跃度优化捞瓶命中质量
- 引入消息中心：回复提醒、系统通知

## UI 迭代记录（欧美审美适配）

- 漂流瓶主流程文案已统一为英文语境，避免中英混杂
- 页面间距与卡片圆角统一，强化轻量、松弛的阅读节奏
- 主题色从高饱和绿色调整为更中性的自然绿，降低视觉疲劳
- 底部导航和内容卡采用一致的弱边框与柔和背景，接近欧美产品常见的克制风格
